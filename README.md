# matibot.github.io

Personal website of Mati Bot, built with Jekyll and hosted on GitHub Pages.

Live site: [https://mati.bot](https://mati.bot)

This page was based on [{ Personal }](https://github.com/PanosSakkos/personal-jekyll-theme) by [Panos Sakkos](https://github.com/PanosSakkos)

## Setup

### Prerequisites

- Ruby 2.6.10 (see `.ruby-version`)
- Bundler gem

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/matibot/matibot.github.io.git
   cd matibot.github.io
   ```

2. Install dependencies:
   ```bash
   bundle install
   ```

### Local Development

Run Jekyll locally:

```bash
bundle exec jekyll serve
```

The site will be available at `http://localhost:4000`

To build the site without running a server:

```bash
bundle exec jekyll build
```

The generated site will be in the `_site/` directory.

### Deployment

This site is automatically deployed via GitHub Pages. Simply push to the `main` branch (or the configured branch) and GitHub Pages will build and deploy the site.

## Configuration

Key configuration options can be found in `_config.yml`:

- Site metadata (title, description, author)
- Navigation pages
- Blog pagination settings
- Social media links
- Career timeline events
- Google Analytics tracking ID

## Dependencies

Main dependencies (see `Gemfile`):

- **Jekyll 4.1.1** - Static site generator
- **jekyll-paginate 1.1.0** - Pagination support for blog posts
- **jekyll-feed 0.17.0** - RSS/Atom feed generation
- **kramdown-parser-gfm 1.1.0** - Markdown parser with GitHub Flavored Markdown support

## Project Structure

- `_config.yml` - Site configuration
- `_layouts/` - Page layouts (index, post, blog, tag, error)
- `_includes/` - Reusable components (header, footer, navigation, etc.)
- `_posts/` - Blog posts (Markdown files with front matter)
- `_data/` - Data files (YAML format)
- `css/` - Stylesheets
- `js/` - JavaScript files
- `img/` - Image assets
- `blog/` - Blog index page
- `tags/` - Tag pages

## Notes

- The site uses Bootstrap 3 for styling
- RSS feed is automatically generated by the `jekyll-feed` plugin at `/feed.xml`
- Posts use Markdown with front matter for metadata
- Tags are automatically generated as pages in the `tags/` directory
